# 코사장 — AI한테 코딩 시키는 사장

## 프로젝트 개요

"코사장" 유튜브/스레드 콘텐츠 제작 프로젝트.
AI 바이브코딩으로 실제 사업 문제를 해결하는 과정을 콘텐츠로 제작.

## 채널 정보

- **채널명**: 코사장 (코딩하는 사장)
- **슬로건**: "AI한테 코딩 시키는 사장"
- **핸들**: @cosajang
- **타겟**: 자동화/바이브코딩에 관심 있는 소상공인, 1인 기업, 예비 창업자

## 포지셔닝

- 노코드 미들웨어(Make, Zapier) 대신 Claude Code 바이브코딩으로 해결
- 코딩을 가르치는 게 아니라, AI한테 코딩을 시키는 법을 보여줌
- 실제 사업자가 시행착오를 공유하는 포지션

### 왜 바이브코딩인가

- Make/Zapier: 월 구독료 부담
- LLM 직접 실행: 비결정성(세무·회계 위험), 토큰비, 느림, 유지보수 어려움
- 바이브코딩: AI가 코드를 짜줌 → 코드는 결정적, 무료, 빠름, 유지보수 가능
- 이 내용은 전용 영상으로 설명하지 않고, 작업 중 자연스럽게 언급

## 기술 스택

- **메인**: Claude Code (바이브코딩)
- **보조**: Airtable (DB), Slack (입출력 인터페이스) 등 필요 시
- **보조 도구 설명**: 깊이 설명하지 않음. 필요 시 외부 튜토리얼 링크 안내

## 콘텐츠 제작 플로우

### 트랙 A — 날것 콘텐츠 (기본)

1. 한 문장 정리 (머릿속 or 메모) — "오늘 뭘 만들고 왜 하는지"
2. 인트로 멘트(30초) → 혼잣말하며 작업 + 화면녹화 → 아웃트로 멘트(30초)
3. 편집 (무음컷 + 자막) → 업로드
4. 스레드 요약 (Claude로 변환)

### 트랙 B — 기획 콘텐츠 (선별적)

1. 5줄 브리프 작성
2. 인트로 나레이션 → 작업 녹화 → 아웃트로 나레이션
3. 편집 (구조 재배치 포함) → 업로드
4. 스레드 요약

### 아웃풋

- Phase 1 (즉시): 풀버전 + 스레드 요약
- Phase 2 (감 잡힌 후): + 숏츠
- Phase 3 (반응 데이터 후): + 요약본 (잘된 영상만)

## 녹화 규칙

- OBS Studio: 화면 + 마이크 동시 녹화
- 혼잣말 스타일 — 시청자에게 설명하듯 의식적으로 말하기
- 후시녹음 없음 — 인트로/아웃트로도 녹화 시작·종료 시 바로 멘트
- 청자 설정: "예전의 나" 또는 "AI 모르는 사장님 친구"
- 막히는 과정을 숨기지 않음 — 시행착오가 핵심 콘텐츠

## 콘텐츠 구조

- 각 영상은 **독립적으로 완결** — 하나만 봐도 가치가 있어야 함
- 시리즈물이되 순서 의존 X — Part 1 안 봐도 Part 3 이해 가능
- 연결은 영상 설명란 링크 + 아웃트로 안내 + 재생목록으로 처리

## 5줄 브리프 템플릿 (트랙 B용)

1. 문제: 뭐가 불편했나 / 왜 이걸 하게 됐나
2. 시도: 어떤 도구로 어떻게 해결하려 했나
3. 과정: 화면 녹화 (여기가 본문)
4. 결과: 전/후 비교 (숫자가 있으면 최고)
5. 한줄 회고: 잘된 점 / 아쉬운 점

## 편집 도구

| 단계 | 내가 하는 일 | 도구 |
|------|-------------|------|
| 녹화 | 버튼 누르기 | OBS |
| 무음 컷 | 확인만 | Recut / DaVinci Auto Cut |
| 자막 | 확인 + 수정 | Whisper / Clova Note |
| 제목·설명·태그 | 선택만 | Claude (트랜스크립트 입력) |
| 썸네일 | 텍스트 교체 | Canva 템플릿 |
| 스레드 변환 | 확인 + 게시 | Claude (트랜스크립트 → 요약) |

## Claude 작업 지시

이 프로젝트에서 Claude에게 요청할 수 있는 작업:
- 트랜스크립트 → 영상 제목/설명/태그 생성
- 트랜스크립트 → 스레드 요약 변환
- 영상 아이디어 브레인스토밍
- 5줄 브리프 작성 도움
- 시리즈 구조 기획
